### Loading Official Normalization Bounds (`feat_min_max.mat`) {#loading-official-normalization-bounds-feat_min_maxmat}

The dataset ships with pre-computed min/max normalization bounds in
`stats/feat_min_max.mat`.\
These are the **official reference values** for min-max scaling and must
be used consistently across train/test.
:::

::: {#f0abf5e8 .cell .code execution_count="83"}
``` python
import scipy.io as sio

# ── Load official normalization bounds ──
mat = sio.loadmat(os.path.join(DATA, 'stats', 'feat_min_max.mat'))

# Build a clean dict: feature -> (min, max)
NORM_FEATS = [f for f in ALL_FEATS if f != 'cpm25'] + ['cpm25']  # all features
norm_bounds = {}
for feat in NORM_FEATS:
    fmin = float(mat[f'{feat}_min'].squeeze())
    fmax = float(mat[f'{feat}_max'].squeeze())
    norm_bounds[feat] = (fmin, fmax)

print(f'Loaded official normalization bounds for {len(norm_bounds)} features:')
print(f'{"Feature":15s} {"Official Min":>15s} {"Official Max":>15s} {"Range":>15s}')
print('─' * 62)
for feat, (lo, hi) in norm_bounds.items():
    print(f'{feat:15s} {lo:15.6e} {hi:15.6e} {hi - lo:15.6e}')
```

## 2A. Official Normalization Bounds vs Computed Statistics {#2a-official-normalization-bounds-vs-computed-statistics}

Compare the **pre-computed min/max bounds** from `feat_min_max.mat` with
the actual data statistics.\
Mismatches may indicate that the official bounds were computed on a
**larger dataset** (e.g., full year) or that our 4-month subset has
outliers outside the normalization range.
:::

::: {#f2658130 .cell .code execution_count="87"}
``` python
# ── Compare official bounds with computed min/max from the 4-month raw data ──
print(f'{"Feature":15s} {"Data Min":>12s} {"Ofcl Min":>12s} {"Match?":>7s}   '
      f'{"Data Max":>12s} {"Ofcl Max":>12s} {"Match?":>7s}   {"OOB%":>7s}')
print('─' * 105)

oob_report = {}
for feat in ALL_FEATS:
    arrs = []
    for m in MONTHS:
        arrs.append(np.load(os.path.join(DATA, 'raw', m, f'{feat}.npy')).astype(np.float64))
    v = np.concatenate(arrs, axis=0)
    dmin, dmax = v.min(), v.max()
    omin, omax = norm_bounds[feat]

    # Check how many pixels fall outside official bounds
    below = np.sum(v < omin)
    above = np.sum(v > omax)
    oob_pct = (below + above) / v.size * 100
    oob_report[feat] = {'below': below, 'above': above, 'pct': oob_pct,
                        'dmin': dmin, 'dmax': dmax}

    min_ok = '  ✅' if abs(dmin - omin) / (abs(omin) + 1e-30) < 0.01 else '  ⚠️'
    max_ok = '  ✅' if abs(dmax - omax) / (abs(omax) + 1e-30) < 0.01 else '  ⚠️'

    print(f'{feat:15s} {dmin:12.4e} {omin:12.4e} {min_ok:>7s}   '
          f'{dmax:12.4e} {omax:12.4e} {max_ok:>7s}   {oob_pct:6.3f}%')
    del v

print()
print('✅ = matches within 1%  |  ⚠️ = data exceeds official bounds')
print('OOB% = percentage of values outside [official_min, official_max]')
```

``` python
def safe_corr(x, y):
    x = np.asarray(x, dtype=np.float64)
    y = np.asarray(y, dtype=np.float64)
    mask = np.isfinite(x) & np.isfinite(y)
    if mask.sum() < 3:
        return np.nan
    x = x[mask]
    y = y[mask]
    sx = x.std()
    sy = y.std()
    if sx == 0 or sy == 0:
        return np.nan
    return float(np.corrcoef(x, y)[0, 1])

m = 'DEC_16'
cpm_ts = np.load(os.path.join(DATA, 'raw', m, 'cpm25.npy')).astype(np.float64).mean(axis=(1, 2))

corr_results = {}

    arr = np.load(os.path.join(DATA, 'raw', m, f'{feat}.npy')).astype(np.float64)
    feat_ts = arr.mean(axis=(1, 2))
    corr = safe_corr(cpm_ts, feat_ts)
```

## 18. Deep Data Scientist Analysis --- Key Findings & Modelling Implications {#18-deep-data-scientist-analysis--key-findings--modelling-implications}

### A. Seasonal Dynamics --- The Core Challenge {#a-seasonal-dynamics--the-core-challenge}

  ----------------------------------------------------------------------------------
  Season             Mean PM2.5      Std    P99     Max      Characteristics
  ------------------ --------------- ------ ------- -------- -----------------------
  **Apr (Summer)**   22.4            27.3   119.0   2534.4   Moderate pollution,
                                                             stable boundary layer

  **Jul (Monsoon)**  15.4            19.9   79.1    843.2    **Lowest** --- wet
                                                             deposition scavenges
                                                             particles

  **Oct              37.2            48.2   182.5   2849.7   Transition, increasing
  (Post-monsoon)**                                           pollution

  **Dec (Winter)**   59.1            79.3   365.8   1537.7   **Highest** --- 4× July
                                                             mean, temperature
                                                             inversions
  ----------------------------------------------------------------------------------

**Critical Insight:** The model must generalize across a **4× range in
mean PM2.5** between seasons. December has **27%** of the variance of
all training data despite being only 25% of the time. This is where RMSE
will be won or lost.

### B. Diurnal Cycle --- The 24-Hour Rhythm {#b-diurnal-cycle--the-24-hour-rhythm}

From the autocorrelation plots:

-   **Strong 24-hour periodicity** visible in all seasons (peak at
    lag=24)
-   April and December show cleaner diurnal cycles
-   July (Monsoon) has **weaker diurnal signal** --- monsoon clouds
    reduce solar heating → flatter PBL cycle
-   October shows **\~12-hour secondary cycle** --- morning and evening
    traffic peaks?

**Model Implication:** The TFNO must learn to capture this \~24h
periodicity. Since the model only sees 10 hours of cpm25 history, it
relies heavily on meteorological features (especially `pblh`) to infer
position in the daily cycle.

### C. Feature Importance Hierarchy {#c-feature-importance-hierarchy}

From correlation analysis (December):

  Rank   Feature          Correlation   Physical Mechanism
  ------ ---------------- ------------- ---------------------------------------
  1      **pblh**         -0.749        Low mixing height → trapped pollution
  2      **t2**           -0.724        Cold temperatures → stable atmosphere
  3      **NMVOC_finn**   -0.631        Biomass burning proxy (spurious?)
  4      **q2**           -0.616        Low humidity → less wet deposition
  5      **NH3**          -0.616        Agricultural emission hotspots
  6      **NOx**          -0.610        Traffic/industrial emission marker
  7      **SO2**          -0.591        Industrial emission signal
  8      **PM25**         -0.585        Primary aerosol emission source
  9      **bio**          -0.476        Biogenic emission marker
  10     **swdown**       -0.435        Solar radiation (day/night proxy)

**Counter-intuitive Findings:**

1.  `rain` shows near-zero (+0.019) correlation despite being a major
    PM2.5 scavenging mechanism because it\'s **70% sparse** --- the
    signal is non-linear and threshold-based.

2.  `NMVOC_finn` ranks #3 with -0.631 correlation --- **likely
    spurious** since it\'s 98% zeros. This correlation may be driven by
    a few extreme biomass burning events coinciding with high PM2.5.

3.  `NMVOC_e` shows exactly zero correlation (-0.000) --- essentially
    noise.

### D. Spatial Heterogeneity --- The Geography Factor {#d-spatial-heterogeneity--the-geography-factor}

From the spatial heatmaps:

**High PM2.5 Hotspots (consistent across seasons):**

-   **Indo-Gangetic Plain** (Northern India) --- visible dark red band
    from Punjab through Bihar
-   **NCR Delhi region** --- persistent hotspot in all seasons
-   **Central India** --- scattered hotspots, likely industrial

**Low PM2.5 Regions:**

-   **Himalayan foothills** --- higher elevation = lower pressure =
    different boundary layer dynamics
-   **Southern India** --- better ventilation, coastal effects
-   **Western coastal regions** --- ocean breeze disperses pollution

### E. Wind Field Patterns --- Transport Mechanisms {#e-wind-field-patterns--transport-mechanisms}

From wind direction plots:

-   **April/December:** Weak, variable winds --- **stagnation favors
    accumulation**
-   **July (Monsoon):** Strong SW monsoon winds --- **dispersal + wet
    deposition**
-   **October:** Transition --- winds weakening, pollution building

**Modelling Insight:** Wind speed matters less than **convergence
zones** where u10/v10 both approach zero. The model should learn these
stagnation patterns.

### F. Emission Features --- The Static Prior {#f-emission-features--the-static-prior}

**Critical Observation:** Emission features (PM25, SO2, NOx, etc.) are:

1.  **Extremely small** (O(10⁻⁸ to 10⁻¹¹) magnitude)
2.  **Temporally constant** within each month --- they\'re gridded
    inventories, not dynamic
3.  **Spatially informative** --- they encode \"where are the sources\"

**Recommendation:** These features act as **static spatial priors** ---
telling the model where pollution originates. Grid-wise normalization is
essential or they\'ll be numerically invisible. \| Metric \| Train (All
4 months) \| Test (2017) \|

### G. Test Data Distribution Shift {#g-test-data-distribution-shift}

\| Mean \| 33.6 µg/m³ \| 33.5 µg/m³ \| \| Max \| 2849.7 µg/m³ \| 2763.8
µg/m³ \| \| Min \| 0.001 µg/m³ \| 0.000 µg/m³ \| \| Distribution \|
Multi-modal (seasonal) \| Unimodal, closest to Oct+Dec \|

**Good news:** Test distribution overlaps well with training --- no
severe covariate shift. **Concern:** Test appears to be from
post-monsoon/winter periods --- model may underperform if
April/July-like samples are in validation.

### H. The Prediction Gap Problem {#h-the-prediction-gap-problem}

From the autocorrelation plots:

-   **t=10 (red line):** Last cpm25 observation --- correlation still
    \~0.6-0.7
-   **t=26 (green line):** First forecast hour --- correlation drops to
    \~0.3-0.4

The model must **extrapolate 16 hours beyond what it can see** in cpm25,
relying entirely on:

1.  Meteorological forecast (which continues for 26h)
2.  Emission priors (static)
3.  Learned dynamics from training

    high-impact)

    or quantile regression
-   The competition metric (Average Domain RMSE) will be heavily
    influenced by extreme events

``` python
# This is critical for understanding what predicts PM2.5 at different forecast horizons

fig, axes = plt.subplots(2, 3, figsize=(18, 10))

# Use December (most challenging month)
m = 'DEC_16'
cpm = np.load(os.path.join(DATA, 'raw', m, 'cpm25.npy')).astype(np.float32)
pblh = np.load(os.path.join(DATA, 'raw', m, 'pblh.npy')).astype(np.float32)
t2 = np.load(os.path.join(DATA, 'raw', m, 't2.npy')).astype(np.float32)
q2 = np.load(os.path.join(DATA, 'raw', m, 'q2.npy')).astype(np.float32)
u10 = np.load(os.path.join(DATA, 'raw', m, 'u10.npy')).astype(np.float32)
v10 = np.load(os.path.join(DATA, 'raw', m, 'v10.npy')).astype(np.float32)

# Domain-average time series
cpm_ts = cpm.mean(axis=(1, 2))
pblh_ts = pblh.mean(axis=(1, 2))
t2_ts = t2.mean(axis=(1, 2))
q2_ts = q2.mean(axis=(1, 2))
wspd_ts = np.sqrt(u10.mean(axis=(1, 2))**2 + v10.mean(axis=(1, 2))**2)

# Features to analyze
features = {
    'cpm25 (autoregression)': cpm_ts,
    'PBLH': pblh_ts,
    'Temperature (t2)': t2_ts,
    'Humidity (q2)': q2_ts,
    'Wind Speed': wspd_ts,
}

# Compute lag correlations (feature at time t correlates with cpm25 at time t+lag)
max_lag = 24  # hours ahead
lags = range(0, max_lag + 1)

colors = ['coral', 'steelblue', 'green', 'purple', 'orange']
ax = axes[0, 0]
for (name, feat_ts), color in zip(features.items(), colors):
    lag_corrs = []
    for lag in lags:
        if lag == 0:
            corr = np.corrcoef(feat_ts, cpm_ts)[0, 1]
        else:
            corr = np.corrcoef(feat_ts[:-lag], cpm_ts[lag:])[0, 1]
        lag_corrs.append(corr)
    ax.plot(lags, lag_corrs, color=color, linewidth=2, label=name)

ax.axhline(0, color='black', linewidth=0.5)
ax.axvline(10, color='red', linestyle='--', alpha=0.7, label='cpm25 input ends')
ax.axvline(16, color='green', linestyle='--', alpha=0.7, label='forecast horizon')
ax.set_xlabel('Forecast Lead Time (hours)')

def partial_autocorr(x, max_lag):
    """Compute partial autocorrelation using OLS regression."""
    n = len(x)
    pacf = [1.0]
    for k in range(1, max_lag + 1):
        # Regress x[k:] on x[k-1:-1], x[k-2:-2], ..., x[0:n-k]
        Y = x[k:]
        X = np.zeros((n - k, k))
        for j in range(1, k + 1):
            X[:, j-1] = x[k-j:n-j]
        # Solve using least squares
        coefs, _, _, _ = lstsq(X, Y, rcond=None)
        pacf.append(coefs[-1])  # Take the coefficient of the most recent lag
    return pacf
```

## 25. Concrete Persistence Baseline RMSE {#25-concrete-persistence-baseline-rmse}

The **persistence model** predicts: *\"the next 16 hours will look
exactly like the last observed hour.\"*\
This gives the **floor RMSE** any serious model must beat. Without this
number, competition scores have no reference.
:::

::: {#2a63ec72 .cell .code execution_count="115"}
``` python
# ── Persistence baseline RMSE: predict last known cpm25 for each of next 16 hours ──
# Strategy: for each sample of T consecutive hours, take the value at hour t,
# then compute RMSE against hour t+k for k=1..16 (this is per-pixel RMSE, averaged over domain).

persist_rmse_per_step = []  # RMSE at each forecast step 1..16
persist_mae_per_step  = []

for m in MONTHS:
    cpm = np.load(os.path.join(DATA, 'raw', m, 'cpm25.npy')).astype(np.float32)
    T, H, W = cpm.shape
    # Slide over all possible windows where we have 10 history + 16 future
    for step in range(1, 17):
        # For each anchor t (t >= 10 to simulate having 10h history), predict t+step
        # "last observed" = cpm25 at t (index 9 within window = last of 10-step input)
        anchors = np.arange(9, T - step)   # hour indices of "last observed"
        last_obs = cpm[anchors]             # (N, H, W)
        future   = cpm[anchors + step]     # (N, H, W)
        diff = last_obs - future
        rmse = np.sqrt((diff ** 2).mean())
        mae  = np.abs(diff).mean()
        persist_rmse_per_step.append(rmse)
        persist_mae_per_step.append(mae)

# Average over months per step
persist_rmse_by_step = np.array(persist_rmse_per_step).reshape(len(MONTHS), 16).mean(0)
persist_mae_by_step  = np.array(persist_mae_per_step).reshape(len(MONTHS), 16).mean(0)

# ── Also compute seasonal breakdown ──
season_persist = {}
for m in MONTHS:
    cpm = np.load(os.path.join(DATA, 'raw', m, 'cpm25.npy')).astype(np.float32)
    T, H, W = cpm.shape
    rmses = []
    for step in range(1, 17):
        anchors = np.arange(9, T - step)
        diff = cpm[anchors] - cpm[anchors + step]
        rmses.append(float(np.sqrt((diff**2).mean())))
```

## 27. Spatial Correlation Decay --- Semivariogram Analysis {#27-spatial-correlation-decay--semivariogram-analysis}

At what **distance (km)** does PM2.5 correlation between two grid cells
drop below 0.5?\
This directly determines the **minimum receptive field** your U-Net /
FNO needs to capture spatial interactions.\
Grid resolution is \~25km, so 8 grid cells ≈ 200km.
:::

::: {#659e3719 .cell .code execution_count="117"}
``` python
# ── Spatial Correlation Decay: sample random pixel pairs at varying distances ──
# Uses December (most spatially structured month)

m = 'DEC_16'
cpm = np.load(os.path.join(DATA, 'raw', m, 'cpm25.npy')).astype(np.float32)  # (T, H, W)
ll  = np.load(os.path.join(DATA, 'raw', 'lat_long.npy'))
lat = ll[:, :, 0]   # (140, 124)
lon = ll[:, :, 1]

T, H, W = cpm.shape

def haversine_km(lat1, lon1, lat2, lon2):
    """Great-circle distance in km between two lat/lon points."""
    R = 6371.0
    dlat = np.radians(lat2 - lat1)
    dlon = np.radians(lon2 - lon1)
    a = np.sin(dlat/2)**2 + np.cos(np.radians(lat1)) * np.cos(np.radians(lat2)) * np.sin(dlon/2)**2
    return 2 * R * np.arcsin(np.sqrt(a))

# Sample N random pixel pairs, compute distance and correlation
rng2 = np.random.default_rng(42)
N_pairs = 3000
i_indices = rng2.integers(0, H, N_pairs)
j_indices = rng2.integers(0, W, N_pairs)
i_indices2 = rng2.integers(0, H, N_pairs)
j_indices2 = rng2.integers(0, W, N_pairs)

distances = []
correlations = []

for k in range(N_pairs):
    r1, c1 = i_indices[k],  j_indices[k]
    r2, c2 = i_indices2[k], j_indices2[k]
    dist = haversine_km(lat[r1, c1], lon[r1, c1], lat[r2, c2], lon[r2, c2])
    ts1 = cpm[:, r1, c1].astype(np.float64)
    ts2 = cpm[:, r2, c2].astype(np.float64)
    if ts1.std() > 0 and ts2.std() > 0:
        r_val = float(np.corrcoef(ts1, ts2)[0, 1])
    else:
        r_val = 0.0
    distances.append(dist)
    correlations.append(r_val)

distances    = np.array(distances)
correlations = np.array(correlations)

# Bin correlations by distance (25km bins matching grid resolution)
bin_edges  = np.arange(0, 2000, 50)   # 0–2000km in 50km bins

for lo, hi in zip(bin_edges[:-1], bin_edges[1:]):
    mask = (distances >= lo) & (distances < hi)
    if mask.sum() > 5:
        bin_corr.append(correlations[mask].mean())
        bin_std.append(correlations[mask].std())
    else:
        bin_corr.append(np.nan)
        bin_std.append(np.nan)

bin_corr = np.array(bin_corr)
bin_std  = np.array(bin_std)
valid    = ~np.isnan(bin_corr)

fig, axes = plt.subplots(1, 2, figsize=(16, 5))

# Plot 1: Scatter + binned mean
ax = axes[0]
ax.scatter(distances[::3], correlations[::3], alpha=0.15, s=8, c='steelblue', label='Pixel pairs')
ax.plot(bin_mids[valid], bin_corr[valid], 'red', linewidth=2.5, marker='o', markersize=5,
        label='Binned mean (50km)')
ax.fill_between(bin_mids[valid],
                (bin_corr - bin_std)[valid],
                (bin_corr + bin_std)[valid],
                alpha=0.2, color='red')
ax.axhline(0.5, color='green', linestyle='--', linewidth=2, label='r = 0.5 threshold')
ax.axhline(0.0, color='black', linewidth=0.8)

# Mark the distance where r drops below 0.5
below_half = bin_mids[valid][bin_corr[valid] < 0.5]
if len(below_half) > 0:
    d_half = below_half[0]
    ax.axvline(d_half, color='green', linestyle=':', linewidth=2)
    ax.text(d_half + 20, 0.55, f'r<0.5\nat {d_half:.0f}km', color='green', fontsize=10, fontweight='bold')

ax.set_xlabel('Distance between pixels (km)', fontsize=12)
ax.set_ylabel('Temporal Correlation of cpm25', fontsize=12)
ax.set_title('Spatial Correlation Decay (Semivariogram)\nDec 2016', fontweight='bold')
ax.legend(fontsize=9)
ax.set_xlim(0, 1800)
ax.set_ylim(-0.2, 1.0)
ax.grid(alpha=0.3)

# Plot 2: Correlation at key distances for architecture sizing
ax = axes[1]
arch_distances = [50, 100, 200, 300, 500, 800, 1200]   # km

for d in arch_distances:
```

## 28. Input Window Asymmetry Analysis {#28-input-window-asymmetry-analysis}

**The core asymmetry:** `cpm25` is available for hours 0--9 (10h), but
all met/emission features are available for hours 0--25 (26h).\
The model must forecast `cpm25` for hours 10--25.

**Key question:** Are the \"future\" met values (hours 10--25)
predictable from the \"input\" met values (hours 0--9)?\
If PBLH at t+16 is predictable from PBLH at t+0..t+9, the model can use
it for guidance even after cpm25 stops.
:::

::: {#c3710d16 .cell .code execution_count="118"}

## 29. PCA / EOF Analysis of PM2.5 Spatial Patterns {#29-pca--eof-analysis-of-pm25-spatial-patterns}

Principal Component Analysis on the space of PM2.5 maps reveals the
**dominant spatial modes** of variability.\
If 2--3 PCs explain \>70% of variance, the problem has **low intrinsic
spatial dimensionality** --- a good sign for grid-to-grid forecasting
architectures.
:::

::: {#61af3486 .cell .code execution_count="119"}
``` python
# ── PCA / EOF Analysis of cpm25 spatial maps ──
# Stack all training months, compute PCA on flattened spatial maps

arrs_pca = []
month_ids = []
for i_m, m in enumerate(MONTHS):
    cpm = np.load(os.path.join(DATA, 'raw', m, 'cpm25.npy')).astype(np.float32)
    arrs_pca.append(cpm)
    month_ids.extend([i_m] * cpm.shape[0])

X_pca = np.concatenate(arrs_pca, axis=0)          # (T_total, H, W)
T_pca, H_pca, W_pca = X_pca.shape
X_flat = X_pca.reshape(T_pca, H_pca * W_pca)      # (T_total, H*W)
month_ids = np.array(month_ids)

# ── Apply official min-max normalization before PCA (model sees normalized data) ──
omin_cpm_pca, omax_cpm_pca = norm_bounds['cpm25']
cpm_range_pca = omax_cpm_pca - omin_cpm_pca
X_normed = np.clip((X_flat.astype(np.float64) - omin_cpm_pca) / cpm_range_pca, 0.0, 1.0).astype(np.float32)

# PCA on NORMALIZED data (what the model actually sees)
X_mean_norm = X_normed.mean(axis=0, keepdims=True)
X_centered_norm = X_normed - X_mean_norm

# Use normalized-centered for PCA (better model-aligned)
X_centered = X_centered_norm

# SVD-based PCA (memory-efficient: use covariance of time × time)
# Since T < H*W (2932 << 17360), solve in time domain
T_cov = X_centered @ X_centered.T / (T_pca - 1)   # (T, T) — smaller matrix
eigenvalues, eigenvectors = np.linalg.eigh(T_cov)

# Sort descending
order = np.argsort(eigenvalues)[::-1]
eigenvalues  = eigenvalues[order]
eigenvectors = eigenvectors[:, order]

# Compute EOFs (spatial patterns): EOF_k = X^T * v_k / ||X^T * v_k||
n_pcs = 8
eofs = []
pcs  = []
for k in range(n_pcs):
    v_k = eigenvectors[:, k]                         # (T,)
    eof_k = X_centered.T @ v_k                       # (H*W,)
    eof_k /= np.linalg.norm(eof_k)                   # unit length
    pc_k = X_centered @ eof_k                        # (T,) scores
    eofs.append(eof_k.reshape(H_pca, W_pca))
    pcs.append(pc_k)

# Explained variance
explained_var = eigenvalues[:n_pcs] / eigenvalues.sum() * 100
cumulative_var = np.cumsum(explained_var)

fig, axes = plt.subplots(3, 4, figsize=(20, 14))

# Row 1+2: First 6 EOFs (spatial patterns)
for k in range(6):
    ax = axes.flat[k]
    vmax = np.abs(eofs[k]).max()
    im = ax.imshow(eofs[k][::-1], cmap='RdBu_r', vmin=-vmax, vmax=vmax)
    ax.set_title(f'EOF {k+1} — {explained_var[k]:.1f}% variance', fontsize=11, fontweight='bold')
    ax.axis('off')
    plt.colorbar(im, ax=ax, fraction=0.046)

# Row 3, col 0-1: Scree plot
ax = axes[2, 0]
ax.bar(range(1, n_pcs+1), explained_var, color='steelblue', alpha=0.8, label='Individual')
ax.plot(range(1, n_pcs+1), cumulative_var, 'red', marker='o', linewidth=2, label='Cumulative')
ax.axhline(70, color='green', linestyle='--', alpha=0.7, label='70% threshold')
ax.axhline(90, color='orange', linestyle='--', alpha=0.7, label='90% threshold')
ax.set_xlabel('Principal Component', fontsize=11)
ax.set_ylabel('Explained Variance (%)', fontsize=11)
ax.set_title('Scree Plot — Variance per EOF', fontweight='bold')
ax.legend(fontsize=8)
```

## 30. Cross-Validation Fold Contamination Check {#30-cross-validation-fold-contamination-check}

`cpm25` has autocorrelation \~0.95 at lag 1h. If you split
training/validation randomly,\
adjacent hours leak: validation samples at time `t` are almost identical
to training samples at `t±1`.\
This **inflates validation scores** and masks real generalization error.

This analysis shows **how large a temporal gap** is needed between folds
to eliminate leakage.
:::

::: {#cc913dfd .cell .code execution_count="120"}
``` python
# ── CV Fold Contamination: Measure how autocorrelation creates leakage ──
# Simulate: train on first 80% of month, validate on last 20%
# Compare error when gap=0 (contaminated) vs gap=G hours (clean)

m = 'DEC_16'
cpm = np.load(os.path.join(DATA, 'raw', m, 'cpm25.npy')).astype(np.float32)  # (T, H, W)
T_cv, H_cv, W_cv = cpm.shape

# Domain-mean ts for leakage analysis
ts_cv = cpm.mean(axis=(1, 2))

# === Part 1: Autocorrelation = raw leakage signal ===
max_gap = 48
autocorr_vals = [1.0]
for lag in range(1, max_gap + 1):
    autocorr_vals.append(float(np.corrcoef(ts_cv[:-lag], ts_cv[lag:])[0, 1]))
autocorr_vals = np.array(autocorr_vals)

# Find gap where autocorr < 0.1 (essentially independent)
ind_gap = next((i for i, v in enumerate(autocorr_vals) if v < 0.1), max_gap)

# === Part 2: Leakage score simulation with varying fold gaps ===
# Split: train = hours [0, split_t), val = hours [split_t + gap, T)
split_t = int(T_cv * 0.8)
gaps = [0, 1, 2, 4, 6, 8, 12, 16, 24, 48]
leakage_scores = []   # correlation between last train sample and first val sample
val_rmse_by_gap = []  # persistence RMSE on val set vs gap

for gap in gaps:
    val_start = split_t + gap
    if val_start >= T_cv - 16:
        leakage_scores.append(np.nan)
        val_rmse_by_gap.append(np.nan)
        continue

    # Leakage: correlation between last train timestep and first val map
    last_train = cpm[split_t - 1].flatten()
    first_val  = cpm[val_start].flatten()
    if last_train.std() > 0 and first_val.std() > 0:
        leak = float(np.corrcoef(last_train, first_val)[0, 1])
    else:
        leak = 0.0
    leakage_scores.append(leak)

    # Persistence RMSE on val set (using last training sample as forecast)
    val_cpm = cpm[val_start:val_start + min(100, T_cv - val_start)]
    last_obs = cpm[split_t - 1]
    reference = np.tile(last_obs[np.newaxis], (len(val_cpm), 1, 1))
    rmse_leak = float(np.sqrt(((val_cpm - reference)**2).mean()))
    val_rmse_by_gap.append(rmse_leak)

# === Part 3: Blocked CV recommendation ===
# For each possible number of time blocks, compute minimum reliable gap
n_blocks = [2, 3, 4, 5, 10]
block_gap_needed = []  # hours gap per block boundary
for nb in n_blocks:
    block_size = T_cv // nb
    needed = ind_gap   # need autocorr < 0.1 between blocks
    usable = block_size - needed
    block_gap_needed.append((nb, block_size, needed, usable, usable / T_cv * 100))

fig, axes = plt.subplots(1, 3, figsize=(18, 5))

# Plot 1: Autocorrelation with highlighted "independent" region
ax = axes[0]
lags = np.arange(max_gap + 1)
ax.bar(lags, autocorr_vals, color='steelblue', alpha=0.7)
ax.axhline(0.1,  color='red',   linestyle='--', linewidth=2, label='r=0.1 (independence threshold)')
ax.axhline(0.5,  color='orange', linestyle='--', linewidth=1.5, label='r=0.5')
ax.axvline(ind_gap, color='red', linestyle=':', linewidth=2)
ax.text(ind_gap + 1, 0.6, f'{ind_gap}h gap needed', color='red', fontsize=10, fontweight='bold')
ax.axvspan(0, ind_gap, alpha=0.1, color='red',  label=f'Contaminated zone (0–{ind_gap}h)')
ax.axvspan(ind_gap, max_gap, alpha=0.1, color='green', label=f'Clean zone ({ind_gap}h+)')
ax.set_xlabel('Gap between Train & Val (hours)', fontsize=11)
ax.set_ylabel('Autocorrelation', fontsize=11)
ax.set_title('Autocorrelation = Raw Leakage Signal\n(How contaminated is a gap of N hours?)', fontweight='bold')
ax.legend(fontsize=8, loc='upper right')

# Plot 2: Leakage correlation by gap size
ax = axes[1]
valid_gaps = [(g, l) for g, l in zip(gaps, leakage_scores) if not np.isnan(l)]
g_vals = [v[0] for v in valid_gaps]
l_vals = [v[1] for v in valid_gaps]
bars = ax.bar(range(len(g_vals)), l_vals, color=['#d32f2f' if v > 0.5 else '#f57c00' if v > 0.1 else '#388e3c' for v in l_vals], alpha=0.85)
ax.bar_label(bars, fmt='%.2f', fontsize=9)

ax.axhline(0.1, color='green', linestyle='--', linewidth=2, label='r=0.1 clean threshold')

# Plot 3: Blocked CV summary table as bar chart
ax = axes[2]
n_b_vals = [b[0] for b in block_gap_needed]

print(f'→ Any CV split needs at least a {ind_gap}-hour gap to avoid leakage\n')
```

``` python
psfc_month = {m: np.load(os.path.join(DATA, 'raw', m, 'psfc.npy')).astype(np.float32) for m in MONTHS}
cpm_month = {m: np.load(os.path.join(DATA, 'raw', m, 'cpm25.npy')).astype(np.float32) for m in MONTHS}

fig, axes = plt.subplots(2, 4, figsize=(20, 9))

# Row 1: seasonal psfc maps
psfc_means = {}
for i, m in enumerate(MONTHS):
    psfc_mean = psfc_month[m].mean(axis=0)
    psfc_means[m] = psfc_mean

    ax = axes[0, i]
    im = ax.imshow(psfc_mean[::-1], cmap='terrain')
    ax.set_title(f"{SEASON_LABELS[m]} psfc mean", fontsize=11, fontweight='bold')
    ax.axis('off')
    plt.colorbar(im, ax=ax, fraction=0.046)

# Row 2: psfc ↔ cpm25 pixel-wise temporal correlation map
psfc_cat = np.concatenate([psfc_month[m] for m in MONTHS], axis=0)   # (Ttot,H,W)
cpm_cat = np.concatenate([cpm_month[m] for m in MONTHS], axis=0)      # (Ttot,H,W)
psfc_flat = psfc_cat.reshape(psfc_cat.shape[0], -1)
cpm_flat = cpm_cat.reshape(cpm_cat.shape[0], -1)

psfc_flat_c = psfc_flat - psfc_flat.mean(axis=0, keepdims=True)
cpm_flat_c = cpm_flat - cpm_flat.mean(axis=0, keepdims=True)
num = (psfc_flat_c * cpm_flat_c).sum(axis=0)
den = np.sqrt((psfc_flat_c**2).sum(axis=0) * (cpm_flat_c**2).sum(axis=0))
corr_flat = np.divide(num, den, out=np.zeros_like(num), where=den > 1e-12)
corr_psfc_map = corr_flat.reshape(H, W).astype(np.float32)

vmax = np.nanpercentile(np.abs(corr_psfc_map), 99)
for i, _m in enumerate(MONTHS):
    ax = axes[1, i]
    im = ax.imshow(corr_psfc_map[::-1], cmap='RdBu_r', vmin=-vmax, vmax=vmax)
    ax.set_title('psfc vs cpm25 corr (same map)', fontsize=11, fontweight='bold')
    ax.axis('off')
    plt.colorbar(im, ax=ax, fraction=0.046)

fig.suptitle('psfc Deep-Dive: Topography-Like Spatial Field + PM2.5 Correlation Structure',
             fontsize=14, fontweight='bold')
plt.tight_layout()
savefig('eda_psfc_spatial_corr.png')
plt.show()

# Simple latitudinal gradient diagnostic
psfc_all_mean = np.mean(np.stack([psfc_means[m] for m in MONTHS], axis=0), axis=0)
north_band = psfc_all_mean[:H//5, :].mean()
south_band = psfc_all_mean[-H//5:, :].mean()

print('\n━━━ psfc SUMMARY ━━━')
print(f'All-season mean psfc: {psfc_all_mean.mean():.1f} Pa')
print(f'North-band mean psfc: {north_band:.1f} Pa')
print(f'South-band mean psfc: {south_band:.1f} Pa')
print(f'North-South difference: {north_band - south_band:.1f} Pa')
print(f'Pixel corr map mean={np.nanmean(corr_psfc_map):.3f}, min={np.nanmin(corr_psfc_map):.3f}, max={np.nanmax(corr_psfc_map):.3f}')
```
